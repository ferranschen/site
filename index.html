Research Projects


MIWA: Mixed-Initiative Web Automation for Better User Control and Confidence

Abstract: In the era of Big Data, web scraping is frequently used by data scientists, domain experts, and end-users to finish time-consuming but essential data collection tasks. However, developing web automation scripts requires familiarity with a programming language and HTML, which remains a key learning barrier for non-expert users. We provide MIWA, a mixed-initiative web automation system that enables users to create web scraping programs by demonstrating what contents they want from the targeted websites. Compared with existing web automation tools, MIWA helps users better understand a generated program and build trust in it by (1) providing a step-by-step explanation of script behavior with visual correspondence to the target website, (2) supporting fine-grained control over web automation, and (3) automatically detecting potential corner cases that are handled improperly by the generated program. Besides, MIWA provides greater autonomy and control to users with features such as undo/redo previous demonstration steps. We conducted a within-subjects user study with 24 participants and compared MIWA with Rousillon, a state-of-the-art web automation tool. Results show that, compared with Rousillon, MIWA reduced the task completion time by half while helping participants gain more confidence on the generated script.

Keywords: Programming by Demonstration, Web Automation, Data Science

Understanding the Coding Behavior of Data Scientists at the Action Level

Abstract: In the era of big data, data scientists play key roles in gaining valuable insights from data in almost every industry and academic field. Despite the widespread use of data science, our understanding of data scientists’ coding behavior remains insufficient. Previous scientific studies on data science practices have been conducted by mining the code from GitHub repositories or Kaggle. However, previous work only provides a snapshot of their coding behavior since most of the collected source codes are presented in the final form, revealing little evidence of their behavior at the action level. Compared to the previous work, we analyzed the source code of all execution histories and error logs to gain a deeper understanding of their behavior. Our work identified the prevalent mistakes made by data scientists, the strategies they use to repair them, and the action-level patterns of data science coding and debugging. Finally, we compared our findings with previous work and demonstrated the implications of more effective data science supporting tools, providing a more nuanced understanding of data scientists.

Keywords: Data Science, Jupyter Notebook, Programming Behavior
